_Points à retenir avant les développements_

1.  Aperçu et compréhension exacte de l'architecture MVC
	1.	Model
	2.	Views
	3.	Controlleur
	4.	Routage 
2.  Prendre en compte le répertoire du schéma de la base de donnée `EcoRide\scripts_DB`
3.  Si tu dois créer de script js:
    1.  Le nom du fichier doit être `script_nome-du-script.js`
    2.  Chemin de fichiers `js`, `EcoRide\public\js\`
    3.  Sauf indication différente, ajouter le fichier au point précédent dans la page `EcoRide\app\Views\templates\default.php`
4.  Si tu dois créer de file css:
    1.  Le nom du fichier doit être `style_nome-du-css.css`
    2.  Chemin de fichiers `css`, `EcoRide\public\css`
    3.  Utiliser les couleurs de l’application, confer `EcoRide\public\css\style_app.css`.
    4.  Sauf indication différente, ajouter le fichier au point précédent dans la page `EcoRide\app\Views\templates\default.php`.
    5.  Ajoute du debug `console.log` afin de suivre le flux
5.  Si il y a quelque chose déjà existante tu dois comprendre si tu peux le réutiliser
6.	Crée un plan d'action avant d'entamer les développements
